# Student Model Training Pipeline

## Overview
This module contains the training pipeline for the **Student Model** as part of the Anchor-Regularized Model Evolution (ARME) project. 

The primary research goal here is to investigate **recursive synthetic learning**. We are training a completely new language model using *only* synthetic data generated by the Frozen Anchor.

**Crucial constraints:**
- The Student Model is initialized with **RANDOM WEIGHTS**.
- No pretrained DistilGPT2 weights are loaded.
- No Anchor weights are copied.
- The model *never* sees WikiText or any human dataset.
- Training data is exclusively sourced from `data/synthetic/generation_1/`.

## Architecture
The pipeline mirrors the Anchor model's architecture to ensure a controlled experiment, but it is entirely separate.

- `student_config.py`: Hyperparameters and configuration.
- `utils.py`: Logging and device utilities.
- `synthetic_dataset.py`: PyTorch Dataset/DataLoader for the Apache Arrow synthetic data.
- `student_model.py`: Model initialization (random DistilGPT2) and tokenizer setup.
- `train_student.py`: Core training loop.
- `evaluate_student.py`: Evaluation and validation loop.
- `save_checkpoint.py`: Checkpoint saving logic.
- `student_statistics.py`: Training history and model statistics.
- `verify_student.py`: Validation script to ensure all constraints (random init, data source, etc.) are met.
- `run_student_training.py`: Main entry point to execute the pipeline.

## Usage

To run the full training pipeline:

```bash
python student/run_student_training.py
```

To verify the pipeline constraints before or after training:

```bash
python student/verify_student.py
```

## Google Colab
This pipeline is designed to execute seamlessly on Google Colab (T4 GPU). It will automatically detect CUDA and utilize Mixed Precision (AMP) if available, falling back to CPU otherwise.
